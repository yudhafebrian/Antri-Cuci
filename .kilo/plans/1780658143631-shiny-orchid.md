# Report Dashboard Page

## Overview
Create a `/report` page showing business statistics: total vehicles, revenue, best-selling packages, trend charts, and calendar performance. Data sourced from completed queue transactions (`stage = 'selesai'`).

## Files to Create/Modify

### 1. New DB functions — `src/lib/db.ts`
Add `loadCompletedQueue(startDate, endDate)` to fetch all `stage = 'selesai'` records within a date range. Supports both Supabase and NeonDB.

### 2. Report utility — `src/lib/reportUtils.ts` (new)
Pure functions to process raw queue data into report metrics:
- `calculateKPIs(data, previousPeriodData)` → totalVehicles, totalRevenue, bestPackage, mostRevenuePackage, growth percentages
- `generateTrendData(data, granularity)` → daily/weekly/monthly/yearly aggregated data for line chart
- `generatePaketBreakdown(data)` → grouped by paket label with count, revenue, percentage (used for both charts)
- `generateCalendarData(data, year, month)` → per-day vehicle count + revenue
- `getDateRange(filter, customStart?, customEnd?)` → resolve filter to start/end dates

### 3. Shared components — `src/components/report/` (new directory)
Extract reusable pieces used by both desktop and mobile views:
- `ReportHeader.tsx` — FIP Autoshop logo + "Report" link, user badge + Logout
- `ReportFilterBar.tsx` — Day | Week | Month | Year | Custom toggle + date range display
- `KPICard.tsx` — single KPI card (icon, label, value, growth %, sparkline)
- `TrendChart.tsx` — line chart for Kendaraan/Revenue trend
- `PaketBarChart.tsx` — horizontal bar chart (reused for both penjualan & revenue)
- `CalendarPerformance.tsx` — color-coded month calendar
- `useReportData.ts` — custom hook: manages filter state, fetches data via `loadCompletedQueue`, computes all derived metrics

### 4. Desktop view — `src/components/report/ReportDesktop.tsx` (new)
Wide layout (max-w-7xl):
- **KPI row**: 4 cards in a row (Total Kendaraan, Total Revenue, Paket Terlaris, Most Revenue)
- **Charts row**: 3 charts side-by-side (Trend line chart, Penjualan bar chart, Revenue bar chart)
- **Calendar**: full-width below charts

### 5. Mobile view — `src/components/report/ReportMobile.tsx` (new)
Stacked layout (full-width, mobile-first):
- **KPI cards**: stacked vertically, 1 per row (4 total)
- **Charts**: stacked vertically, full-width each
- **Calendar**: compact grid, smaller day cells, scrollable if needed
- Filter bar: horizontal scroll for Day/Week/Month/Year/Custom buttons

### 6. Report page — `src/pages/ReportPage.tsx` (new)
Thin wrapper that detects viewport width (e.g. `window.innerWidth >= 768`) and renders either `ReportDesktop` or `ReportMobile`. Re-renders on resize. Handles auth check — redirects to `/admin` if not logged in.

### 7. Route — `src/App.tsx`
Add `<Route path="/report" element={<ReportPage />} />`. ReportPage handles its own auth check (redirect to `/admin` if not logged in).

### 8. Navigation link — `src/pages/AdminPage.tsx`
Add a "Report" button/tab in the admin header navigation linking to `/report`.

## Design Decisions

- **KPI Cards**: 4 cards total — Total Kendaraan, Total Revenue, Paket Terlaris, Most Revenue (mostRevenuePackage). "Total Paket Terjual" dihapus sesuai feedback user.
- **Responsive strategy**: Separate files for desktop (`ReportDesktop.tsx`) and mobile (`ReportMobile.tsx`).
- **Charts**: Use `recharts` directly (LineChart, BarChart, Bar, XAxis, YAxis, Tooltip, ResponsiveContainer).
- **Colors**: Match screenshot — blue for Kendaraan, green for Revenue, orange/amber for "Sangat Ramai", green-tint for "Ramai", blue-tint for "Normal", white for "Sepi".
- **Data source**: Only `stage = 'selesai'` records count as completed transactions. Revenue = sum of `harga`. Previous period comparison for growth %.
- **Filter logic**:
  - Day: single selected date
  - Week: Mon–Sun of selected week
  - Month: full calendar month (default: current month)
  - Year: full calendar year
  - Custom: two date inputs for start/end
- **Calendar thresholds**: Sepi (0–20), Normal (21–40), Ramai (41–60), Sangat Ramai (60+) — configurable constants.
- **Paket grouping**: Map raw `paket` field values to the 5 display categories: Regular Wash, Premium Wash, Wash Detail, Coating & Wax, Lainnya — based on the group names in `constants.ts`.
